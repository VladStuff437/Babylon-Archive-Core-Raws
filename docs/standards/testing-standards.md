# Стандарты тестирования — Babylon Archive Core

## Цель
Обеспечить предсказуемое качество разработки на 150 сессий через единый тестовый контур.

## Структура тестов
- `SessionNNN/SessionNNNTests.cs` — тесты конкретной сессии
- `Smoke/Block01SmokeTests.cs` — быстрые проверки целостности фундамента

## Базовые правила
- Каждый тест должен быть детерминирован
- Тесты не используют сетевые вызовы и внешние сервисы
- Один тест проверяет один контракт
- Имена тестов: `What_Condition_ExpectedResult`

## Категории
- `Smoke`: быстрые проверки (<1s на тест)
- `Invariant`: проверки INV-xxx
- `Session`: проверка deliverables сессии

## Минимальные требования к новой сессии
- Добавить минимум один тест в `SessionNNN`
- Обновить checklist сессии
- При добавлении нового инварианта: добавить/обновить invariant test

## CI требования
- `dotnet test BabylonArchiveCore.sln --configuration Release --no-build`
- Падение любого smoke/invariant теста блокирует merge
