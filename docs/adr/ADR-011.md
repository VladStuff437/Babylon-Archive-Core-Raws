# ADR-011: Схема данных игры v1

**Статус**: Accepted
**Дата**: Сессия 011
**Контекст**: ADR-010, Block 02 start

## Решение
Ввести контракт `Session011DataContract` как базовую модель игровых данных v1 с обязательными полями идентификации сейва, версии схемы, seed, режима и world tick.

## Альтернативы
1. Хранить только runtime-снимок без явного контракта: отклонено из-за низкой прозрачности.
2. Перенести контракт полностью в Content: отклонено, так как runtime сериализация должна быть типобезопасной.

## Риски
- Дублирование полей между runtime и content schema.
- Риск рассинхронизации при ручном обновлении.

## Связи
- Реализует: INV-016
- Зависит от: ADR-009, ADR-010
- Влияет на: S012-S015
