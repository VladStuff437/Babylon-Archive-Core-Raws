# ADR-017: Runtime State Manager Baseline

**Статус**: Accepted
**Дата**: Сессия 017
**Контекст**: ADR-016

## Решение
Ввести RuntimeStateManager как единый runtime-компонент синхронизации режима, тика и dirty-state поверх WorldStateReader.

## Альтернативы
1. Оставить состояние только в WorldStateReader: отклонено.
2. Управлять состоянием на уровне UI: отклонено, нарушает слой Runtime.

## Риски
- Возможна избыточная синхронизация при частых переходах режимов.
- Потребуется расширение метрик state drift в следующих сессиях.

## Связи
- Реализует: INV-022
- Зависит от: ADR-016
- Влияет на: S018+
