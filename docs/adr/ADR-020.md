# ADR-020: Control Profiles v1 Consolidation

**Статус**: Accepted
**Дата**: Сессия 020
**Контекст**: ADR-019

## Решение
Ввести контракт Session020ControlProfilesContract и runtime-компонент ControlProfileResolver для унификации active/fallback маршрута профилей управления.

## Альтернативы
1. Оставить fallback-логику только в ActionMapCatalog: отклонено.
2. Управлять профилями только через UI-конфиг без runtime-проверки: отклонено.

## Риски
- Рост числа профилей может усложнить ручную поддержку profileChain.
- Для мульти-девайс режима потребуется дополнительный слой override.
