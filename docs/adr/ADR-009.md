# ADR-009: CI/CD Strategy

**Статус**: Accepted
**Дата**: Сессия 009
**Контекст**: ADR-007, ADR-008, GitHub Actions, .NET 8

## Решение

Использовать GitHub Actions для CI pipeline с тремя стадиями:

### Pipeline stages

```
push/PR → [Build] → [Test] → [Analyze]
```

1. **Build**: `dotnet build --configuration Release --no-restore` — компиляция всех проектов
2. **Test**: `dotnet test --no-build --configuration Release` — запуск xUnit тестов
3. **Analyze**: Будущее — Roslyn analyzers, invariant checks (P-009)

### Trigger rules
- **push** на `main` — полный pipeline
- **pull_request** на `main` — полный pipeline
- Workflow вручную (`workflow_dispatch`)

### Constraints
- .NET 8 SDK
- Ubuntu-latest runner
- `TreatWarningsAsErrors=true` уже есть — warnings = build failure
- Test failures = pipeline failure

### Будущие расширения (deferred)
- Code coverage reporting (P-009)
- Invariant runtime checks в CI
