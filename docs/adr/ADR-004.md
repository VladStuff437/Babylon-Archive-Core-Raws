# ADR-004: Формальная структура проектов и слоёв

**Статус**: Принято
**Дата**: Сессия 004
**Контекст**: ADR-001, ADR-003, INV-002

## Решение

Создать отдельные .csproj для каждого архитектурного слоя, объединённые в solution:

### Структура проектов
```
BabylonArchiveCore.sln
├── src/
│   ├── BabylonArchiveCore.Core/          ← Domain models, contracts, state definitions
│   │   └── BabylonArchiveCore.Core.csproj (no dependencies)
│   ├── BabylonArchiveCore.Runtime/       ← Game loop, generation, combat, AI
│   │   └── BabylonArchiveCore.Runtime.csproj → Core
│   ├── BabylonArchiveCore.UI/            ← HUD, screens, input, rendering
│   │   └── BabylonArchiveCore.UI.csproj → Runtime, Core
│   └── BabylonArchiveCore.Content/       ← Data files, schemas
│       └── BabylonArchiveCore.Content.csproj → Core
├── tests/
│   └── BabylonArchiveCore.Tests/
│       └── BabylonArchiveCore.Tests.csproj → Core, Runtime, UI, Content
└── tools/
```

### Зависимости (enforced через ProjectReference)
| Проект | Зависит от | Запрещено |
|--------|-----------|-----------|
| Core | — (ничего) | Runtime, UI |
| Runtime | Core | UI |
| UI | Runtime, Core | — |
| Content | Core | Runtime, UI |
| Tests | All (для тестирования) | — |

### Ответственности слоёв
- **Core**: Доменные модели, интерфейсы контрактов, WorldState definition, enums, DTO
- **Runtime**: Реализация контрактов, GameLoop, генерация, combat, AI, save/load
- **UI**: Рендеринг, HUD, ввод, адаптеры к Runtime через IViewStateProvider/IInputHandler
- **Content**: JSON/YAML данные, схемы, шаблоны. Только Core-типы для схем.

## Альтернативы
1. **Monolithic single project**: Отклонено — нет compile-time enforcement зависимостей
2. **NuGet packages per layer**: Отклонено — overhead для активной разработки
3. **Folders-only separation**: Отклонено — convention без enforcement

## Риски
- Circular reference при неаккуратном добавлении — митигация: CI + тесты
- Overhead от ProjectReference при сборке — митигация: минимален для 4 проектов
