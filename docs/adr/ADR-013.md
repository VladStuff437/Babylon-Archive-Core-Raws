# ADR-013: Политика версионирования схем

**Статус**: Accepted
**Дата**: Сессия 013
**Контекст**: ADR-011, ADR-012

## Решение
Ввести explicit контракт версии схемы (`Session013SchemaVersionContract`) и формальные правила совместимости: current, minimum compatible, channel, migration policy.

## Альтернативы
1. Хранить только одно поле schemaVersion: отклонено из-за недостатка контекста совместимости.
2. Не различать каналы stable/preview: отклонено, мешает управляемым экспериментам.

## Риски
- Усложнение пайплайна релизов при расширении каналов.
- Требуется дисциплина синхронизации migration policy с кодом.

## Связи
- Реализует: INV-018
- Зависит от: ADR-012
- Влияет на: все будущие data migration сессии
