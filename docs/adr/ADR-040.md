# ADR-040: Mission Runtime Orchestration

## Status
Accepted

## Context
Session 040 introduces an explicit runtime orchestration layer for mission progression on top of MissionDefinition and TransitionEvaluator.

## Decision
- Introduce Session040 mission-runtime contract/schema/migration/serializer/profile.
- Add MissionRuntimeEngine and MissionRuntimeState to manage start/advance/completion flow.
- Add runtime/mission/smoke tests covering visited-node tracking and balance-driven path selection.

## Consequences
- Mission execution flow is explicit, testable, and deterministic.
- Runtime can evolve mission progression behavior without mutating core mission contracts.
