# ADR-008: Code Standards

**Статус**: Accepted
**Дата**: Сессия 008
**Контекст**: ADR-007, Directory.Build.props, existing code

## Решение

Принять .editorconfig как единый enforcement инструмент для стиля кода. Дополнить человекочитаемым `coding-standards.md`.

### Naming Conventions
- **Типы** (class, struct, interface, enum): PascalCase
- **Интерфейсы**: `I` + PascalCase (`IWorldStateReader`)
- **Методы, Properties**: PascalCase
- **Параметры, локальные переменные**: camelCase
- **Приватные поля**: `_camelCase`
- **Константы**: PascalCase (`MaxRetries`)
- **Enum members**: PascalCase

### Formatting
- Indentation: 4 spaces (no tabs)
- Braces: Allman style (new line)
- Max line width: 120 chars (soft limit)
- UTF-8, LF line endings

### C# Idioms (C# 12 / .NET 8)
- File-scoped namespaces обязательны
- `var` когда тип очевиден, explicit когда неочевиден
- `Nullable=enable` (уже в Directory.Build.props)
- Pattern matching предпочитается `is` + cast
- Records для immutable value objects
- Primary constructors для DI

### Enforcement
- `.editorconfig` — IDE + build-time
- `TreatWarningsAsErrors=true` — уже в Directory.Build.props
- Roslyn analyzers (будущее — P-008) — compile-time lint

## Альтернативы
1. **StyleCop только**: Отклонено — .editorconfig более универсален
2. **Без формализации**: Отклонено — дрейф неизбежен на 150 сессиях
3. **Stricter rules (все `var` запрещены)**: Отклонено — снижает читаемость

## Риски
- Retro-apply на существующий код → митигация: постепенное приведение, не всё сразу
- Слишком строгие правила замедляют → митигация: severity=suggestion для non-critical
