# ADR-003: Boundary Contracts между слоями

**Статус**: Принято
**Дата**: Сессия 003
**Контекст**: INV-002, ADR-001, ADR-002 (Pillar P6)

## Решение

Формализовать границы между слоями через **контрактные интерфейсы** в Core:

### Граница Core ↔ Runtime
| Контракт | Направление | Описание |
|----------|-------------|----------|
| `IWorldStateReader` | Core → Runtime | Только чтение состояния мира |
| `IWorldStateMutator` | Core → Runtime | Мутация через command pattern |
| `ICommandDispatcher` | Core → Runtime | Диспетчеризация команд изменения |
| `IGenerator` | Core → Runtime | Контракт генерации (seed-based) |
| `IContentLoader` | Core → Runtime | Загрузка и валидация контента |

### Граница Runtime ↔ UI
| Контракт | Направление | Описание |
|----------|-------------|----------|
| `IViewStateProvider` | Runtime → UI | Read-only проекция состояния для UI |
| `IInputHandler` | Runtime → UI | Обработка пользовательского ввода |
| `IUIEventBus` | Runtime → UI | Уведомления UI о событиях |

### Правила передачи данных
1. **Core→Runtime**: Интерфейсы + DTO (immutable records). Runtime реализует интерфейсы Core.
2. **Runtime→UI**: ViewState DTO (read-only snapshots). UI **никогда** не получает ссылки на мутабельные объекты.
3. **Cross-layer вызовы**: Только через определённые контракты. Прямые `new Runtime.SomeClass()` из UI запрещены.

### Enforcement Mechanism
- **Compile-time**: Directory.Build.props + Roslyn analyzer (будущее) запрещают обратные ссылки
- **Convention**: Все boundary interfaces в `Core/Contracts/` namespace
- **Test-time**: Тесты проверяют, что public API слоя не возвращает внутренние типы другого слоя

## Альтернативы
1. **Reflection-based enforcement**: Отклонено — сложно, хрупко, медленно
2. **Separate NuGet packages per layer**: Отклонено — оверкилл для монорепы на старте
3. **No formal contracts (convention only)**: Отклонено — дрейф неизбежен на 150 сессиях

## Риски
- Интерфейсы могут стать «god interfaces» → митигация: ISP (Interface Segregation)
- Command pattern может добавить boilerplate → митигация: codegen в tools/
- ViewState может рассинхрониться с WorldState → митигация: автоматическая проекция
