# ADR-022: Interaction Contour — Camera, Interactions, Inventory, Combat, AI (S022)

## Status
Accepted

## Context
Block 03 requires a minimal gameplay contour: camera modes, object interactions, inventory, player state, combat cycle (Direct→AOE→Status→Resolve), tab-target/esc-cancel input, auto-attack, and basic AI perception + state machine.

## Decision
- Camera: `ICameraController` + `CameraController` wrapping `CameraStateManager`.
- Interactions: `IInteractable` protocol in Core, `InteractionExecutor` in Runtime.
- Inventory: `InventoryItem` + `InventoryManager` with capacity/stack limits.
- PlayerState: base attributes (HP, Stamina, Level, XP) with range validation.
- Combat: `StatusEffect`/`StatusEffectManager` for tick effects; `CombatSystem` with ordered phases; `AutoAttackController` with safe stop.
- Input: `CombatInputHandler` with Tab-target / Esc-cancel.
- AI: `PerceptionSystem` (radius detection), `AIStateMachine` (Idle/Patrol/Chase/Attack/Flee/Dead).
- Schema, serializer, migration, control profile, events, taxonomy extended.

## Consequences
- Full gameplay contour is testable end-to-end.
- Enables S023 (inventory v1 consolidation) and subsequent combat/AI refinements.
