# ADR-044: Deterministic Mission Fallback Routing

## Status
Accepted

## Context
Session 044 hardens fallback mission generation so invalid mission graphs are redirected through a deterministic, reproducible fallback flow.

## Decision
- Introduce Session044 fallback contract/schema/migration/serializer/profile.
- Normalize fallback reason codes deterministically before fallback mission construction.
- Extend Runtime/Missions/Generation tests with fallback golden behavior checks.

## Consequences
- Equivalent validation error sets produce identical fallback mission metadata.
- Fallback path remains reproducible and testable across runs.
