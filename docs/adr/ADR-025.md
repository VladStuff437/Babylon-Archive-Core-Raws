# ADR-025: Status System Consolidation

## Status
Accepted

## Context
Session 025 stabilizes status effect mechanics for combat and runtime state transitions.

## Decision
- Extend status effects with category, stack count, and max stacks.
- Support stack merge and targeted removal in `StatusEffectManager`.
- Add session artifacts (contract, schema, migration, serializer, tests).

## Consequences
- Status effects become reusable for buffs/debuffs and deterministic tick simulation.
- Runtime can manage status lifecycle explicitly.
