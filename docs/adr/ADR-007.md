# ADR-007: Repository Structure Convention

**Статус**: Accepted
**Дата**: Сессия 007
**Контекст**: ADR-004, INV-009, текущая файловая структура

## Решение

Фиксируем стандартную структуру репозитория Babylon Archive Core:

```
/
├── src/                              # Исходный код
│   ├── BabylonArchiveCore.Core/      # Доменный слой (0 зависимостей)
│   │   ├── Contracts/                # Boundary interfaces
│   │   ├── Invariants/               # Invariant contracts
│   │   ├── Logging/                  # Logging contracts
│   │   ├── Models/                   # Domain models (WorldState, etc.)
│   │   └── README.md
│   ├── BabylonArchiveCore.Runtime/   # Слой исполнения (→ Core)
│   │   ├── Invariants/               # Invariant checkers
│   │   ├── Logging/                  # Log service impl
│   │   └── README.md
│   ├── BabylonArchiveCore.UI/        # Презентация (→ Runtime)
│   │   ├── Logging/                  # UI log rendering
│   │   └── README.md
│   └── BabylonArchiveCore.Content/   # Данные и схемы (→ Core)
│       └── README.md
├── tests/                            # Тесты
│   └── BabylonArchiveCore.Tests/     # xUnit test project (→ все слои)
│       ├── Session001/               # Тесты по сессиям
│       ├── Session002/ ... Session010/
│       └── README.md
├── docs/                             # Документация
│   ├── adr/                          # Architecture Decision Records
│   │   ├── ADR-NNN.md
│   │   ├── INDEX.md
│   │   └── TEMPLATE.md
│   ├── architecture/                 # Архитектурные документы
│   │   ├── invariants.md
│   │   ├── pillar-module-mapping.md
│   │   └── layer-responsibilities.md
│   ├── checklists/                   # Чеклисты внедрения
│   ├── decisions/                    # decision-log.md
│   ├── masterplan/                   # Сессии + roadmap
│   ├── reports/                      # Closure reports
│   └── standards/                    # Стандарты разработки
├── tools/                            # Утилиты (скрипты, анализаторы)
├── MasterPlan/                       # MasterPlan.md (source of truth)
├── BabylonArchiveCore.sln            # Solution file
├── Directory.Build.props             # Глобальные build свойства
├── .gitignore
├── .editorconfig                     # (S008)
└── README.md
```

### Конвенции именования
- **Проекты**: `BabylonArchiveCore.{Layer}` — PascalCase
- **Папки внутри проекта**: PascalCase по подсистеме (Contracts/, Models/, Logging/)
- **Файлы .cs**: Одно имя = одному типу. `ClassName.cs`
- **Docs**: kebab-case для md файлов, ADR-NNN для ADR
- **Тесты**: `Session{NNN}/Session{NNN}Tests.cs`

### Запреты
- Никаких `.cs` файлов в root
- Никаких проектных файлов вне `src/` или `tests/`
- Никаких binary/build артефактов в repo

## Альтернативы
1. **Flat structure**: Отклонено — не масштабируется на 150 сессий
2. **Monorepo с packages**: Отклонено — overkill для одного продукта
3. **Feature-based folders**: Отклонено — конфликтует с layer-based архитектурой (ADR-001)

## Риски
- Разрастание docs/ → митигация: стандарт + INDEX файлы
- Session-based тесты могут дублироваться → митигация: рефакторинг в Block 02+

## Связи
- Реализует: INV-012
- Зависит от: ADR-004 (project structure), ADR-001 (3-layer)
- Влияет на: все последующие сессии (файловые конвенции)
