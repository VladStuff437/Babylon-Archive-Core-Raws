# ADR-005: Invariant Validation Strategy

**Статус**: Принято
**Дата**: Сессия 005
**Контекст**: INV-001..INV-009, ADR-001..ADR-004

## Решение

Создать **runtime invariant validation framework** для автоматической проверки архитектурных инвариантов:

### Архитектура
```
Core/Invariants/
├── IInvariantChecker.cs       ← контракт проверки
├── InvariantResult.cs         ← результат проверки (pass/fail + details)
├── InvariantSeverity.cs       ← Critical / Warning / Info
└── InvariantRegistry.cs       ← реестр всех чекеров

Runtime/Invariants/
├── WorldStateSSOTChecker.cs   ← INV-001
├── LayerDependencyChecker.cs  ← INV-002
├── BoundaryContractChecker.cs ← INV-008
└── ProjectReferenceChecker.cs ← INV-009
```

### Правила
1. **IInvariantChecker** определяется в Core — контракт без реализации
2. **Concrete checkers** в Runtime — имеют доступ к состоянию
3. **Запуск**: в тестах (всегда), при debug-сборках (опционально), в CI (обязательно)
4. **Результат**: InvariantResult содержит Id, Severity, Pass/Fail, Description
5. **Интеграция с LogService**: нарушения логируются как `LogLevel.Error`

### Enforcement
- Debug builds: `#if DEBUG` секции для runtime проверок
- Tests: InvariantChecker запускается в AfterEach/TearDown
- CI: Отдельный step `dotnet test --filter Category=Invariant`

## Альтернативы
1. **Только тесты (без framework)**: Отклонено — нет runtime detection
2. **Assertions everywhere**: Отклонено — не структурировано, трудно поддерживать
3. **External tool (ArchUnit.NET)**: Отклонено — dependency, overkill на старте

## Риски
- Debug performance overhead — митигация: `#if DEBUG` guards
- False positives при неполном state — митигация: skip checks when uninitialized

## Связи
- Реализует: все INV-*, INV-010
- Зависит от: ADR-001..ADR-004
- Влияет на: все последующие сессии (invariant checks run on every change)
