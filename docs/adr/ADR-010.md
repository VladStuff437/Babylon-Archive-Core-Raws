# ADR-010: Baseline Test Contour

**Статус**: Accepted
**Дата**: Сессия 010
**Контекст**: ADR-005, ADR-009, тесты S001–S009

## Решение

Зафиксировать базовый тестовый контур Block 01 как обязательный слой контроля качества.

### Test contour
1. **Session tests**: `tests/BabylonArchiveCore.Tests/SessionNNN/` — артефакты и инварианты каждой сессии
2. **Smoke tests**: быстрые проверки целостности фундамента (структура, ADR, CI, инварианты)
3. **CI gate**: запуск `dotnet test` на каждом push/PR в `main`

### Test categories
- `Smoke`: критические проверки целостности фундамента
- `Invariant`: проверки INV-xxx
- `Session`: регрессионные проверки конкретной сессии

### Правила
- Smoke тесты должны быть быстрыми и без внешних зависимостей
- Новая сессия добавляет минимум один Session-test
- Инварианты должны иметь автоматическую проверку (напрямую или через session suite)

## Альтернативы
1. **Только интеграционные тесты**: Отклонено — слишком медленно для базового CI
2. **Только unit-тесты без smoke**: Отклонено — нет быстрых проверок целостности проекта
3. **Отложить тестовый контур на позже**: Отклонено — риск регрессий в Block 02+

## Риски
- Рост времени тестов с числом сессий → митигация: выделение smoke-набора
- Дублирование тестов между сессиями → митигация: consolidation в следующих блоках
