# ADR-042: Dead-End Safety and Mission Runtime Persistence

## Status
Accepted

## Context
Session 042 formalizes dead-end safety for mission graphs and introduces snapshot persistence for mission runtime state.

## Decision
- Introduce Session042 dead-end contract/schema/migration/serializer/profile.
- Add DeadEndValidator and enforce non-terminal transition safety rules.
- Add MissionRuntimeSnapshot and MissionRuntimePersistence save/restore flow.

## Consequences
- Non-terminal dead ends are blocked before mission start.
- Mission runtime state can be persisted and restored without losing progression semantics.
