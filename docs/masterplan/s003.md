# Сессия 003 — Границы v1

## Контекст
Сессии 001–002 зафиксировали трёхслойную архитектуру (ADR-001), 7 Core Pillars (ADR-002) и 7 инвариантов. Теперь необходимо формализовать **границы между слоями** — определить контрактные интерфейсы, правила передачи данных и механизмы enforcement.

## Ограничения
- Pillar P6: Layered Architecture Contract — однонаправленные зависимости enforced
- INV-002: Core ← Runtime ← UI (обратные зависимости запрещены)
- Pillar P3: State Sovereignty — UI не мутирует WorldState
- Код в src/ — только контрактные интерфейсы и boundary marker'ы
