# Сессия 006 — ADR-контракт

## Контекст
Сессии 001–005 создали 5 ADR'ов, но сам процесс создания ADR не формализован. Необходимо зафиксировать **ADR как контракт** — шаблон, жизненный цикл, правила ревью и механизм привязки ADR к инвариантам и коду.

## Ограничения
- Все существующие ADR (001–005) должны быть совместимы с новым шаблоном
- ADR не должны устаревать молча — lifecycle management обязателен
- Привязка: ADR → INV → Code → Tests
