# Сессия 005 — Системные инварианты

## Контекст
Сессии 001–004 определили инварианты INV-001..INV-009 в документации. Теперь необходимо **кодифицировать** их — создать runtime-валидацию инвариантов, которая может запускаться в тестах и при debug-сборках.

## Ограничения
- Инварианты — блокирующие правила. Нарушение = дефект.
- Валидация инвариантов не должна влиять на Release performance
- Должна быть интегрирована с LogService (S001)

## Входные артефакты
- `docs/architecture/invariants.md` — 9 инвариантов
- `src/BabylonArchiveCore.Core/Contracts/` — boundary interfaces
- `src/BabylonArchiveCore.Core/Logging/` — log system
- `.csproj` files — project structure
