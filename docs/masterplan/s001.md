# Сессия 001 — Продуктовое видение

## Контекст
Первая сессия проекта **Babylon Archive Core** — RPG с roguelike-элементами, боевой системой, дипломатией и исследованием Архива.
Проект стартует с нуля: рабочая область содержит только MasterPlan.

## Ограничения
- WorldState — единственный источник правды для изменяемого состояния мира (Правило 1)
- Слои Core → Runtime → UI: однонаправленные зависимости, без обратных связей (Правило 2)
- Контент и миссии валидируются автоматически (Правило 3)
- Каждая сессия завершается обновлением ADR, тестов и отчёта (Правило 4)
- Автосборка/запуск только по команде Мастера (Правило 6)

## Архитектурные слои
| Слой | Пространство имён | Ответственность |
|------|-------------------|-----------------|
| Core | `BabylonArchiveCore.Core` | Доменные модели, контракты, состояние (WorldState, PlayerState, MissionDefinition, MissionNode, ArchiveAddress) |
| Runtime | `BabylonArchiveCore.Runtime` | Игровой цикл, боевая система, ИИ, генерация, миссии, события, аудио, сцены |
| UI | `BabylonArchiveCore.UI` | HUD, экраны, ввод, доступность, локализация, debug-оверлеи |

## Иерархия адресации Архива
```
Sector → Hall → Module → Shelf → Tome → Page
```
Каждый уровень — детерминированно генерируется из seed.
